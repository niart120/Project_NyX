# ポケモンコード（内部種族コード）マッピング

> **出典**: [改造ポケモン制作資料 Wiki — ポケモンコード](https://wikiwiki.jp/pokemonhack/%E3%83%9D%E3%82%B1%E3%83%A2%E3%83%B3%E3%82%B3%E3%83%BC%E3%83%89)  
> **用途**: `selphy_rewards.md` §3 のポケモン決定処理で使用する、  
>  内部コード (`species_code`) ↔ 全国図鑑番号 ↔ ポケモン名 の対応表  
> **有効範囲**: 第3世代 (RSE / FRLG) 内部データ。コード 0x0001〜0x019B (1〜411)

---

## 1. 概要

第3世代のポケモン内部データでは、全国図鑑番号とは異なる **内部種族コード** でポケモンを管理する。
アキホおねだりの乱数計算式 `species_code = (rand % 411) + 1` で得られるのはこの内部コードであり、
全国図鑑番号やポケモン名に変換するにはマッピングテーブルが必要となる。

### コード体系

| 範囲 | 10進 | 内容 |
|------|------|------|
| 0x0000 | 0 | ？？？？？（欠番） |
| 0x0001〜0x00FB | 1〜251 | カントー・ジョウト (No.001〜251) ※図鑑番号と一致 |
| 0x00FC〜0x0114 | 252〜276 | **ダミー** (？) — 図鑑上は No.387〜411 に割当 |
| 0x0115〜0x019B | 277〜411 | ホウエン (No.252〜386) ※図鑑番号と不一致 |
| 0x019C | 412 | タマゴ（特殊エントリ） |

> **ポイント**: コード 1〜251 は全国図鑑番号と完全一致する。
> コード 277〜411 はホウエンポケモンだが図鑑番号順ではなく、内部配置順になっている。
> コード 252〜276 (25個) はダミーエントリで、有効なポケモンではない。

## 2. 完全マッピングテーブル

### 凡例

| 列 | 説明 |
|-----|------|
| **内部コード** | 16進表記の内部種族コード |
| **10進** | 内部コードの10進表記。`(rand % 411) + 1` の結果値 |
| **図鑑No.** | 全国図鑑番号。ダミーの場合は本来の図鑑番号が入るが実体がない |
| **ポケモン名** | 日本語名。ダミーは「？」 |

### 3.3 ダミー判定

```python
# ダミーコードの範囲
DUMMY_CODE_START = 252   # 0x00FC
DUMMY_CODE_END = 276     # 0x0114

def is_dummy(species_code: int) -> bool:
    """内部コードがダミーエントリかどうかを判定する。"""
    return DUMMY_CODE_START <= species_code <= DUMMY_CODE_END
```

## 4. アキホおねだりとの関連

### 4.1 ポケモン決定での使用箇所

`selphy_rewards.md` §3 の処理フローにおいて:

1. `species_code = (rand >> 16) % 411 + 1` で内部コードを取得
2. `is_dummy(species_code)` でダミー判定 → ダミーならリトライ
3. `INTERNAL_TO_NATIONAL[species_code]` で全国図鑑番号に変換
4. 図鑑登録済みかを（ゲーム内で）チェック → 未登録ならリトライ

### 4.2 rand 値域とダミー確率

- `rand % 411` の結果は 0〜410 → `+1` で 1〜411
- ダミー範囲: 252〜276 → 25個 / 411 ≈ **6.08%**
- フェーズ1（100回リトライ）では、毎回約6%の確率でダミーに当たる

### 4.3 逆引き: 全国図鑑番号 → 内部コード

フレーム検索時に「特定のポケモンが選ばれるフレーム」を探す場合、
全国図鑑番号から内部コードへの逆引きが必要になる。

```python
# 起動時に逆引きテーブルを構築
NATIONAL_TO_INTERNAL: dict[int, int] = {
    nat: idx
    for idx, nat in enumerate(INTERNAL_TO_NATIONAL)
    if idx > 0  # 欠番 (0) を除外
}
```
