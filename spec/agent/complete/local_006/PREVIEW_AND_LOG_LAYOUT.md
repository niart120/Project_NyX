# GUI 外観再設計: プレビューとログ配置詳細仕様

> **文書種別**: 詳細仕様。プレビュー、右ツールログ、プレビュー下マクロログ、仮想コントローラとの位置関係を定義する。
> **親仕様**: `spec\agent\local_006\WINDOW_SIZE_AND_PANEL_LAYOUT.md`
> **対象モジュール**: `src\nyxpy\gui\panes\preview_pane.py`, `src\nyxpy\gui\panes\log_pane.py`, `src\nyxpy\gui\panes\virtual_controller_pane.py`, `tests\gui\`

## 1. 配置方針

中央列は上にプレビュー、下にプレビュー下マクロログを置く。右列はツールログを縦長に表示する。左列下部の仮想コントローラは状態バー側に接地させ、プレビュー下マクロログは仮想コントローラ領域へ掛けない。

```text
+----------------------+--------------------------+----------------+
| マクロ一覧パネル     | プレビュー               | ツールログ     |
|                      | 固定 16:9                | 常時表示       |
+----------------------+--------------------------+                |
| 仮想コントローラ     | プレビュー下マクロログ   |                |
+----------------------+--------------------------+----------------+
| 状態バー: 接続状態 / 実行状態                                   |
+------------------------------------------------------------------+
```

## 2. プレビュー

| プリセット | 固定サイズ |
|------------|------------|
| HD | `640x360` |
| FullHD | `1280x720` |
| WQHD | `1600x900` |
| 4K | `2560x1440` |

`PreviewPane` は 16:9 を維持し、表示領域に合わせて引き伸ばさない。余剰領域は余白として扱う。

入力フレームがプレビュー固定サイズより大きい場合は、固定サイズへ縮小表示する。入力フレームが固定サイズより小さい場合は、16:9 を維持して拡大表示する。中央クロップは行わない。

## 3. ログ種別

| 種別 | 位置 | 目的 |
|------|------|------|
| マクロログ | 中央列のプレビュー直下 | ユーザーが実行中に読むログ |
| ツールログ | 右列 | デバッグログ、ツールログ、補助情報 |

マクロログとツールログは混在させない。初期実装では同じ `LogPane` 系 widget を使ってよいが、表示フィルタや sink 登録は分けられる設計にする。デバッグ表示切り替えは pane ごとに独立させる。

プレビュー下マクロログの領域は初期状態で表示する。折りたたみは将来拡張とし、初期実装の受け入れ条件には含めない。

## 4. 状態バー

状態バーは接続状態と実行状態の常時表示位置である。

表示例:

```text
準備完了 / 映像: USB Video Device 接続中 / シリアル: COM6 接続中
実行中 / 映像: window capture 接続中 / シリアル: 未接続
```

状態バーに出す文言は短くする。詳細な接続先変更や再接続は設定ダイアログへ誘導する。
